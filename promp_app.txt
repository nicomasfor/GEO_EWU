PROMPT MAESTRO — GEO BUSINESS INTELLIGENCE ENGINE

Quiero que construyas una aplicación web avanzada para descubrimiento masivo de negocios por zona y sector. La primera versión estará orientada únicamente a hostelería y obtendrá datos exclusivamente desde OpenStreetMap, pero la arquitectura debe quedar preparada para añadir más fuentes en el futuro.

El proyecto debe construirse como un monorepo desplegable en Coolify mediante Docker Compose. La base de datos principal será Convex. El backend auxiliar puede ser Python, preferiblemente FastAPI, para ejecutar tareas pesadas de búsqueda, normalización, enriquecimiento, jobs y comunicación con fuentes externas. El frontend debe tener una experiencia visual muy potente, en modo oscuro por defecto, con estética de herramienta avanzada tipo inteligencia geoespacial, inspirada en sistemas modernos de análisis de datos para entidades de poder, sin copiar marcas ni diseños concretos.

- La única fuente inicial de datos será OpenStreetMap mediante Overpass API o una capa equivalente basada en OSM.
- La app debe estar preparada para añadir fuentes futuras como Foursquare, Yelp, directorios oficiales, webs públicas, datasets abiertos o conectores privados.

CATEGORÍAS INICIALES DE HOSTELERÍA PARA OSM
============================================================

El sector Hostelería debe buscar tipos de establecimientos con mesas o consumo presencial, orientado inicialmente a restaurantes, bares y cafeterías.

Mapear desde OpenStreetMap etiquetas como:
- amenity=restaurant
- amenity=cafe
- amenity=bar
- amenity=pub
- amenity=fast_food
- amenity=food_court
- amenity=ice_cream
- tourism=hotel con restaurant si hay datos relacionados
- shop=bakery solo si tiene consumo/cafetería cuando esté indicado
- cuisine=* como metadato adicional

No incluir de primeras:
- supermercados
- tiendas de alimentación puras
- vending machines
- gasolineras salvo que tengan restaurante explícito
- alojamientos sin restaurante
- negocios no relacionados con consumo presencial
